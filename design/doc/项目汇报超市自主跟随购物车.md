# 项目汇报:超市自主跟随购物车

## 一.前期调研

**行业背景：**目前超市里还缺乏智能化的购物车，大部分场景下需要顾客自己用手去推动，较为费力；如果可以实现自主跟随的购物车，解放顾客的双手，可以提升顾客的消费体验。

同时在自主跟随的购物车在调度和管理方面更加智能，在顾客使用完毕后可以自主归位，减少工作人员的介入，提升运作效率。

**可行性支撑：**OpenBot开源项目已具备完整的手机遥控、自动导航及数据采集框架；已有YOLO、MobileNet-SSD等行人检测模型。

## 二.需求分析

**项目定位：** 开发一台成本可控的超市自主跟随购物车（概念验证模型），提供可动手实践的一体化平台，为端侧深度学习等算法提供真实载体。

**核心功能需求：**

* **自主跟随：** 能够在室内平坦路面上稳定跟随指定行人，需包含行人检测与跟踪功能。
* **避障绕行：**具备障碍物检测能力，在超市的场景中，可以通过减速或绕行避开静态的货架，以及动态的其他非指定行人。
* 具备基本的路线规划 、运动控制及人机交互功能。

**约束条件需求**： 主要部件的整体成本必须控制在1500元以内 。

## 三. 初步的项目方案

**整体架构：** 基于 OpenBot 软硬件架构进行模块化设计。利用安卓手机作为上位机（提供摄像头感知与 AI 算力），底层结合微控制器（如 Arduino）驱动低成本小车底盘。

**具体任务拆解与实施：**

* **硬件搭建：** 采购底盘、微控制器、摄像头等，组装并测试基础运动控制。  

* **算法开发：** 使用 Python、Android Studio 等工具链 ，部署轻量级行人检测模型，实现目标跟踪与跟随控制。

* **系统集成：** 整合感知、决策、控制模块 。  

* **测试优化：** 在室场地验证跟随稳定性，优化性能参数 。

**项目预期出口：**

* 一台完整可运行的自动跟随小车实物。
* 全套模块化开源代码。
